Global Secrets allow you to define and share secrets across all registered clusters. Native Kubernetes secrets that reference these global secrets can then be created and Loft will synchronize the secret's data with the global secret.

:::note Space secrets, but not Virtual Cluster secrets
Global secrets are not synchronized to secrets created within virtual clusters, however project secrets are. To use a global secret to manage secret data in virtual clusters, you can first create a project secret that is synchronized by a global secret.
:::

## Create a Global Secret
To create a global secret follow these steps:
<Flow id="secret-create-global-secret">
    <Step>Navigate to the <NavStep>Global Secrets</NavStep> view using the menu on the left</Step>
    <Step>Click on the <Button>Add Global Secret</Button> button</Step>
    <Step>Enter a name for the secret by clicking on <Label>my-secret</Label></Step>
    <Step>Enter a decritpion for the secret by clicking on <Label>Enter a description for this object...</Label></Step>
    <Step>Click on the <NavStep>Data</NavStep></Step>
    <Step>Enter the secret data in the <Label>Secret Data</Label> input</Step>
    <Step>Enter the namespace for the secret in the <Label>Secret Data</Label> input</Step>
    <Step>Click on the <NavStep>Access</NavStep></Step>
    <Step>Use the <Button>+ Rule</Button> to add new access rules for the shared secret</Step>
    <Step>Under the <Label>Users & Teams</Label> column, select the user or team you'd like to grant access to the shared secret</Step>
    <Step>Under the <Label>Permissions</Label> column, select the verb to enable for the user or team</Step>
    <Step>Click on the <Button>Save Changes</Button> button once your changes are complete</Step>
</Flow>

To use this global secret, create a secret in the namespace where you need the secret data, taking care to add the labels:
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
  namespace: my-space
  labels:
    loft.sh/sharedsecret-namespace: loft
    loft.sh/sharedsecret-name: my-secret
type: Opaque
```
