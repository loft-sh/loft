You can either use curl or kubectl to create a new Cluster Connect.

<Tabs
    defaultValue="kubectl"
    values={[
      {label: 'kubectl', value: 'kubectl'},
      {label: 'curl', value: 'curl'},
    ]}>
  <TabItem value="kubectl">


Create a file `object.yaml` with the following contents:
```yaml
apiVersion: management.loft.sh/v1
kind: ClusterConnect
metadata:
  creationTimestamp: null
  name: my-cluster
spec:
  adminUser: my-user
  clusterTemplate:
    metadata:
      creationTimestamp: null
    spec:
      config: {}
    status: {}
  config: |-
    apiVersion: v1
    kind: Config
    clusters:
    - cluster:
    ...
status: {}

```

Then create the Cluster Connect `my-cluster` with:
```bash
kubectl create -f object.yaml 
```


  </TabItem>
  <TabItem value="curl">


Create a file `object.yaml` with the following contents:
```yaml
apiVersion: management.loft.sh/v1
kind: ClusterConnect
metadata:
  creationTimestamp: null
  name: my-cluster
spec:
  adminUser: my-user
  clusterTemplate:
    metadata:
      creationTimestamp: null
    spec:
      config: {}
    status: {}
  config: |-
    apiVersion: v1
    kind: Config
    clusters:
    - cluster:
    ...
status: {}

```

Run the following curl command to create a new Cluster Connect `my-cluster`:
```bash
curl -s -X POST --insecure \
     "https://$LOFT_DOMAIN/kubernetes/management/apis/management.loft.sh/v1/clusterconnect" \
     --data-binary "$(cat object.yaml)" \
     -H "Content-Type: application/yaml" \
     -H "Authorization: Bearer $ACCESS_KEY"
```
